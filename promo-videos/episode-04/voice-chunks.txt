01|大家好，我是老丁。OneNote Markdown 第四集，我们用实际操作讲清内容怎样进来、怎样带走，以及怎样调整字体、字号、代码行号和界面语言。
02|先看 GitHub README。当前版本导入本地的点 M D 或点 Markdown 文件，不直接读取网页链接。先下载 README 或克隆仓库，再点击“导入 Markdown”选择文件；单个文件上限十兆。
03|插件用 UTF 八读取内容，解析标题、列表和代码，然后把渲染结果追加到当前页面。注意是追加，不会清空原内容。这里是实际导入后的页面，内容仍可继续编辑、标注和整理。
04|点击“导出 Markdown”，当前页面会转换回 Markdown，并用页面标题生成默认文件名。保存结果是 UTF 八无 BOM 的点 M D，可以提交 GitHub，也能交给其他 Markdown 编辑器继续处理。
05|只想把内容粘贴到聊天窗口、文档工具或代码仓库，就点击“复制 Markdown”，或者按 F 八。插件同样先转换整个当前页面；看到成功提示，结果已经写入系统剪贴板。
06|设置里可以分别指定正文、代码和数学字体，正文字号与代码字号也能独立调整。配置保存在主题设置文件中，保存后重新渲染即可应用；重要页面建议重启 OneNote 后再做最终渲染。
07|打开“显示代码行号”并重新渲染，代码块会按行编号，适合教程和代码审阅；普通笔记关闭它更简洁。LaTeX 图片渲染也在同一区域，可以分别控制。
08|界面语言支持自动跟随系统、中文和 English。保存后部分文字会立即变化；功能区和已经打开的窗口要完整刷新，建议重启 OneNote。语言设置不会翻译笔记内容。
09|记住三条路径：本地 README 用“导入”，需要文件用“导出”，临时带走用“复制”或 F 八。项目已开源，GitHub 搜索 oldding 的 OneNote Markdown，国内用户访问 nexanote 点 C N 下载，两种渠道都可以。
